"Mobile First" approach: develop for mobile browsers. Start small, grow big. "The more a design hinges on viewing requirements, the less flexible it becomes."

"Progressive enhancement" is the idea of designing from a most common denominator - what most users can see - and building out. Itâ€™s also the fast track to mobile-first design.

Our current breakpoints:

  • Default: Any window greater than 300px and less than 480px
  • Small: Any window greater than or equal to 480px
  • Medium: Any window greater than or equal to 720px
  • Large: Any window greater than or equal to 960px
  • XL: Any window greater than or equal to 1800px
  • Content max-width: 1200px. Varies by site

Developing for retina:

  • Use media queries for high-res CSS styles
  • Have two versions of CSS sprites: standard dimensions and @2x
    • Sprites have largely been replaced with custom icon fonts (see fontastic.me) or using SVGs w/ PNG fallbacks
  • Serve up larger html-embedded images with JavaScript
    • Picturefill, Retina.js
    • More pixels = better display. But, more pixels doesn't necessarily have to mean 4 or 5 times more data. More detail can be accomplished with less kilobytes.
    • Downside: Having to maintain two versions of graphical assets and images.
  • You can just use the same image assets for both standard and high-res screens
    • Downside: Users may have to download larger files than they actually need

Designing for retina:

  • Where possible, use vectors and Smart Objects.
  • Design like you normally would, then open "modify image size" and edit your pixel dimensions to 200 percent to get the 2x view.
